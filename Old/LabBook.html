Inertial Measurement Unit (IMU) and Kalman Filters

Sensors

IMU measures: angular rotation, force and magnetic field, measured using a: gyroscope, accelerometer and a magnetometer respectively. Each of these measurements has serious issues: the accelerometer is easy to confuse with shaky motion but is accurate with slow steady motion, the magnetometer will lose its pointing accuracy over time but is accurate in shaky motion. These problems are rectified using sensor fusion.

Kalman Filters

Kalman filters are a form of optimal estimation algorithm. The measurements from gyroscope, accelerometer and magnetometer are each prone to drift over time. However by combining these sensors it is possible for our estimates to converge towards the real value. The state observer x^\hat{x}x^ represents the estimated state vector from a set of measurements (zzz) which include noise.

1 Recursive Average filters

The estimated state (in this case the mean state) is evaluated as:

x^k=z1+z2+z3+...+zkk(1.1)\hat{x}_k = \frac{z_1+z_2+z_3+...+z_k}{k} \tag{1.1}x^k​=kz1​+z2​+z3​+...+zk​​(1.1)

Which can be written recursively as:
x^k=(k−1k)x^k−1+zkk(1.2)\hat{x}_k = (\frac{k-1}{k})\hat{x}_{k-1} + \frac{z_k}{k} \tag{1.2}x^k​=(kk−1​)x^k−1​+kzk​​(1.2)
which is more efficent to compute.

Let α≔k−1k\alpha \coloneqq \frac{k-1}{k}α:=kk−1​ equation (1.2) can be rewritten as:

x^k=αx^k−1+(1−α)zk(1.3)\hat{x}_k = \alpha \hat{x}_{k-1} + (1 - \alpha) z_k \tag{1.3}x^k​=αx^k−1​+(1−α)zk​(1.3)

This is an example of an recursive (average) filter.


figure 1.1: A noisy signal with mean 25 and standard deviation 0.5.
The filter is efficient and converges to the correct value very quickly. Due to random variation the filter fluctuates around the correct value. For obvious reasons this filter will not work for a constantly varying signal.

2 Moving Average filters

The moving average r^n\hat{r}_nr^n​ is used to remove noise over a constantly varying signal. Here nnn represents the index of the moving average and kkk represents the window size.

x^n=zn−k+1+zn−k+2+...+znk(2.1)\hat{x}_n = \frac{z_{n-k+1} + z_{n-k+2}+...+z_n}{k} \tag{2.1}x^n​=kzn−k+1​+zn−k+2​+...+zn​​(2.1)

x^n\hat{x}_nx^n​ can be written recursively as:

x^n=x^n−1+zn−zm−kk(2.2)\hat{x}_n = \hat{x}_{n-1} + \frac{z_n - z_{m-k}}{k} \tag{2.2}x^n​=x^n−1​+kzn​−zm−k​​(2.2)

The moving average lags behind the true signal but has roughly the right shape.

3 Low pass filter

A low pass filter is an improvement on the moving average filter it allows low frequencies to pass through but filters out high frequencies. Noise is usually high frequency. Below is an example of a first order low pass filter.

x^k=αx^k−1+(1−α)zk0<α<1(3.1)\hat{x}_k = \alpha \hat{x}_{k-1} + (1 - \alpha) z_k \quad 0<\alpha<1 \tag{3.1}x^k​=αx^k−1​+(1−α)zk​0<α<1(3.1)
Looks like expression (1.3) except here α\alphaα is a free parameter to be chosen. It is also true that:
x^k−1=αx^k−2+(1−α)zk−10<α<1(3.2)\hat{x}_{k-1} = \alpha \hat{x}_{k-2} + (1 - \alpha) z_{k-1} \quad 0<\alpha<1 \tag{3.2}x^k−1​=αx^k−2​+(1−α)zk−1​0<α<1(3.2)
Combining these equations helps to overcome some problems associated with the moving average since more distant terms disappear exponentially quickly:

x^k=α2x^k−2+α(1−α)zk−1+(1−α)zk0<α<1(3.3)\hat{x}_k = \alpha^2 \hat{x}_{k-2} + \alpha(1-\alpha) z_{k-1} + (1-\alpha)z_k \quad 0<\alpha<1 \tag{3.3}x^k​=α2x^k−2​+α(1−α)zk−1​+(1−α)zk​0<α<1(3.3)
Due to the restriction on alpha larger nnn means greater weighting on x^n\hat{x}_nx^n​ since α(1−α)≤1−α\alpha(1-\alpha)\leq 1-\alphaα(1−α)≤1−α. Previous data gets weighted exponentially less.

The low pass filter has a smaller delay compared to the moving average filter. However the low pass filter didn't remove as much noise as the moving average filter did.


Figure 3.2: Low pass filter with α=0.8\alpha = 0.8α=0.8
Here the delay is less significant but the filter doesn't remove as much noise as it did when α=0.9\alpha = 0.9α=0.9, since the moving average is more easily influenced by more recent measurements.


Figure 3.3 Low pass filter with α=0.95\alpha = 0.95α=0.95
Here the delay is more significant since results are given more similar weightings, hence figure 3.3 looks most similar to the moving average filter but with a smaller delay. The choice of α\alphaα represents a tradeoff between a noisy signal and a delayed signal, meaning there is an optimal choice of α\alphaα exists ensuring that the overall effects of both are minimized.

4 Kalman filters

Kalman filter is essentially a low pass filter with a dynamically changing α\alphaα. States can be different usually measurements. External input zkz_kzk​ (measurement), final output x^k\hat{x}_kx^k​ (estimate), final output x^k−\hat{x}^-_kx^k−​ (prediction), error covariance PkP_kPk​ (estimate), error covariance Pk−P_k^-Pk−​ (prediction) system model A, H, Q and R all others are used for internal computation.

A prediction is a forecast of the next state based on the previous state and the mathematical model whereas a estimate is an update of the predicted state once new measurements have been taken.

Estimation step

Computation of an estimate (where xkx_kxk​ is a n×1n\times1n×1 column vector) can be written as:
x^k=x^k−+Kk(zk−Hx^k−)(4.1)\hat{x}_k = \hat{x}^-_k + K_k(z_k-H\hat{x}^-_k) \tag{4.1}x^k​=x^k−​+Kk​(zk​−Hx^k−​)(4.1)

rewritten as
x^k=(I+KkH)x^k+Kkzk(4.2)\hat{x}_k = (\mathbb{I} + K_kH)\hat{x}_k + K_kz_k \tag{4.2}x^k​=(I+Kk​H)x^k​+Kk​zk​(4.2)

Letting H=IH = \mathbb{I}H=I and α=1−Kk\alpha= 1-K_kα=1−Kk​ a first order low pass filter is recovered similar to (3.1).

x^k=αx^k−1−+(1−α)zk(4.3)\hat{x}_k = \alpha \hat{x}^-_{k-1} + (1 - \alpha) z_k \tag{4.3}x^k​=αx^k−1−​+(1−α)zk​(4.3)

Kalman filter estimation process is like a low pass filter with dynamically changing α\alphaα (or equivalently gain KkK_kKk​) The gain gets updated in time according to KkK_kKk​:

Kk=Pk−HT(HPk−HT+R)−1(4.4)K_k = P^-_kH^T(HP^-_kH^T+R)^{-1} \tag{4.4}Kk​=Pk−​HT(HPk−​HT+R)−1(4.4)

The error covariance (a measure of the inaccuracy of the estimate) is updated as:
Pk=Pk−−KkHPk−(4.5)P_k = P^-_k-K_kHP^-_k \tag{4.5}Pk​=Pk−​−Kk​HPk−​(4.5)
The true state is xk∼N(x^k,Pk)x_k \sim N(\hat{x}_k, P_k)xk​∼N(x^k​,Pk​) x^\hat{x}x^ is the mean and PkP_kPk​ is the variance. R and H are only in the estimation step

Prediction step

Using the model x^k+1\hat{x}_{k+1}x^k+1​ can be predicted from the state at time tkt_ktk​, x^k+1−=Ax^k\hat{x}^-_{k+1} = A\hat{x}_{k}x^k+1−​=Ax^k​, where A is a n×nn\times nn×n matrix. Similarly Pk−=APkAT+QP^-_k = AP_kA^T + QPk−​=APk​AT+Q. Where QQQ is the process noise or state transition noise. This means (4.1) can be rewritten as:

x^k=Ax^k−1+Kk(zk−HAx^k−1)(4.6)\hat{x}_k = A \hat{x}_{k-1} + K_k(z_k-HA \hat{x}_{k-1}) \tag{4.6}x^k​=Ax^k−1​+Kk​(zk​−HAx^k−1​)(4.6)
A and Q are only in the prediction step

Linear model

Whereas the low pass filter passes x^k−1\hat{x}_{k-1}x^k−1​ directly between time steps tk−1t_{k-1}tk−1​ and tkt_{k}tk​ the kalman filter predicts the next step before a measurement is carried out to produce a new estimate.

The kalman filter deals with the linear state model. Where the state xk+1=Axk+wkx_{k+1} = Ax_k + w_kxk+1​=Axk​+wk​ and the measurement zk+1=Hxk+vkz_{k+1} = Hx_k + v_kzk+1​=Hxk​+vk​.

  • xkx_kxk​ is the state variable (n×1n\times1n×1 column vector)
  • zkz_kzk​ is the measurement (m×1m\times1m×1 column vector).
  • AAA is the state transition matrix (n×nn\times nn×n matrix) which represents the model.
  • HHH is the state to measurement matrix (m×nm \times nm×n matrix).
  • wkw_kwk​ is the linear process noise (n×1n\times1n×1 column vector)
  • vkv_kvk​ is the linear measurement noise (m×1m\times1m×1 column vector)
  • QQQ is the covariance matrix of wkw_kwk​ (n×nn\times nn×n diagonal matrix) i.e. how much the true state is expected to deviate from the predictions made by the state transition model. Large Q the filter assumes the model is less reliable and vice versa.
  • RRR is the covariance matrix of vkv_kvk​ (m×mm\times mm×m diagonal matrix). RRR tells the kalman filter how much to "trust" the measurments compared to model predictions. Large R means the measurments are trusted less so the filter relies more on its predictions.
  • x0x_0x0​ is the initial state estimate. provided at the start of the estimation process (n×1n \times 1n×1 column vector)
  • P0P_0P0​ is the initial error estimate
